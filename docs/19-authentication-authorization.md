# 19. 認証・認可編

## 目的

Strutsアプリケーションでログイン機能、セッション管理、権限チェックを設計する方法を学びます。

## 認証と認可

| 用語 | 意味 |
|---|---|
| 認証 | 利用者が誰かを確認すること |
| 認可 | 利用者がその操作をしてよいか確認すること |

## ログイン処理の流れ

```text
ログイン画面
  ↓
LoginAction
  ↓
AuthService
  ↓
UserRepository
  ↓
セッションへログインユーザー保存
```

## セッションに保存する情報

保存してよい例:

- userId
- userName
- role

保存すべきでない例:

- パスワード
- クレジットカード番号
- 不要な個人情報

## 認証Interceptor

```text
ログイン済み → Action実行
未ログイン → login画面へ遷移
```

## 認可設計

| ロール | 操作 |
|---|---|
| ADMIN | 全機能利用可能 |
| STAFF | 会員一覧・編集可能 |
| USER | 自分の情報のみ閲覧可能 |

## セキュリティ上の注意

- パスワードはハッシュ化する
- 平文パスワードを保存しない
- ログイン失敗回数を制限する
- セッション固定攻撃対策を行う
- ログアウト時にセッションを破棄する
- CSRF対策を行う
