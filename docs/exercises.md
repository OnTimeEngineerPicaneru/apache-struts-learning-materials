# 演習問題

## 演習1: Struts の概要

### 問1

Apache Struts の説明として正しいものを1つ選べ。

A. JavaScript のフロントエンドフレームワークである  
B. Java の Web アプリケーション向け MVC フレームワークである  
C. データベース管理システムである  
D. ビルドツールである  

答え: B

---

## 演習2: MVC

### 問2

Struts において Controller に近い役割を持つものはどれか。

A. JSP  
B. Action  
C. CSS  
D. SQL  

答え: B

---

## 演習3: Action の責務

### 問3

大規模開発において Action に書くべきでない処理はどれか。

A. Service の呼び出し  
B. 処理結果の返却  
C. SQL を直接実行する処理  
D. 画面表示用データの保持  

答え: C

---

## 演習4: セキュリティ

### 問4

Struts 1.x について正しい説明はどれか。

A. 現在も新規開発に推奨される  
B. 公式に EOL となっている  
C. JavaScript 専用である  
D. Maven では使用できない  

答え: B

---

## 演習5: 実装課題

以下を実装せよ。

1. `/hello` にアクセスするとメッセージを表示する
2. `/goodbye` にアクセスすると別のメッセージを表示する
3. メッセージ生成処理を Service クラスに分離する
4. JSP に直接 Java コードを書かない
5. struts.xml に Action を追加する

## 演習6: 設計課題

ユーザー管理機能を Struts で設計せよ。

必要な画面:

- ユーザー一覧
- ユーザー登録
- ユーザー編集
- ユーザー削除確認
- ユーザー削除完了

作成するもの:

1. URL 一覧
2. Action 一覧
3. JSP 一覧
4. Service 一覧
5. 画面遷移表
