# 05. Apache Struts のセキュリティ

## Struts 学習でセキュリティが重要な理由

Apache Struts は過去に重大な脆弱性が報告されてきたフレームワークです。

そのため、Struts を学習する際は、機能だけでなく安全な使い方も学ぶ必要があります。

## 基本方針

- 最新の安定版を使用する
- EOL バージョンを使用しない
- 不要なプラグインを使わない
- 入力値検証を必ず行う
- ファイルアップロードを慎重に扱う
- エラーメッセージに内部情報を出さない
- 依存ライブラリの脆弱性を定期的に確認する

## Struts 1.x について

Struts 1.x は公式に End of Life となっています。

新規開発で使用してはいけません。

既存システムで Struts 1.x を利用している場合は、以下を検討します。

- Struts 2 / 6系への移行
- Spring MVC / Spring Boot への移行
- 外部公開範囲の制限
- WAF の導入
- 依存ライブラリの棚卸し
- 脆弱性診断
- 段階的リプレース

## 入力値検証

入力値は必ずサーバー側で検証します。

悪い例:

```text
JavaScript でチェックしているからサーバー側ではチェックしない
```

良い例:

```text
画面側でもチェックし、サーバー側でも必ずチェックする
```

## ファイルアップロード

ファイルアップロード機能では、以下を確認します。

- 拡張子
- MIME タイプ
- ファイルサイズ
- 保存先パス
- 実行権限
- ファイル名のサニタイズ
- ウイルススキャン

## エラー処理

本番環境では、スタックトレースや内部クラス名を画面に表示しません。

## 依存関係の確認

Maven を使用している場合は、以下のようなツールで依存関係を確認します。

```bash
mvn dependency:tree
```

脆弱性確認には OWASP Dependency-Check などの利用を検討します。

## 本番運用前チェックリスト

- [ ] Struts が最新の安定版である
- [ ] EOL バージョンを使っていない
- [ ] 入力値検証を実装している
- [ ] ファイルアップロード制限がある
- [ ] エラー画面に内部情報を出していない
- [ ] ログに個人情報を出していない
- [ ] 依存ライブラリを棚卸ししている
- [ ] 管理画面に認証・認可がある
- [ ] 不要なデバッグ設定を無効化している
