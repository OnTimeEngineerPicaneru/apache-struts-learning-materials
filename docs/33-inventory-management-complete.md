# 33. 総合演習③ 在庫管理システム

## 目的

業務システムで頻出する在庫管理を題材に、Strutsで複数テーブルを扱う実務的なアプリケーションを設計・実装します。

## 完成要件

- 商品マスタ管理
- 倉庫マスタ管理
- 在庫一覧
- 入庫処理
- 出庫処理
- 在庫移動
- 在庫履歴
- 在庫数アラート
- 管理者ログイン
- 操作ログ

## テーブル例

| テーブル | 説明 |
|---|---|
| products | 商品 |
| warehouses | 倉庫 |
| stocks | 現在庫 |
| stock_movements | 在庫移動履歴 |
| users | 利用者 |
| operation_logs | 操作ログ |

## 在庫処理の注意点

- 入庫、出庫、移動はトランザクションで処理する
- 出庫時に在庫不足をチェックする
- 在庫履歴を必ず残す
- 現在庫と履歴の整合性を保つ
- 同時更新を考慮する

## 出庫処理の流れ

```text
出庫依頼
  ↓
在庫確認
  ↓
stocks更新
  ↓
stock_movements登録
  ↓
operation_logs登録
  ↓
commit
```

## 排他制御

在庫管理では同時更新が問題になります。

検討する方式:

- 悲観ロック
- 楽観ロック
- versionカラム
- SQLの `for update`
